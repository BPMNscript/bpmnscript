# @bpmn-script/language

The Langium language package for BPMNscript. Provides the grammar, parser, AST, validator, and language server used by the VS Code extension and the CLI.

## In plain terms

This package knows the BPMNscript _language_ itself: what counts as valid syntax, and how to turn source text into a structured tree the rest of the toolchain can work with.

You write the language's rules once, in a single grammar file (`src/bpmn-script.langium`). [Langium](https://langium.org/) reads that grammar and generates three things for you: a **parser** (text â†’ tree), the **AST** types (the tree's shape, as TypeScript interfaces), and a **language server** that powers the in-editor experience â€” syntax highlighting, autocompletion, and the red squiggles under mistakes. None of that generated code is checked in; `npm run build` regenerates it from the grammar.

The only logic written by hand on top of Langium is the **validator** (`src/bpmn-script-validator.ts`): the structural rules a parser alone can't express, such as "every process needs a start event" or "a gateway's default branch can't carry a condition." Langium already reports purely syntactic errors and unresolved references; the validator adds the BPMN-specific rules.

## Purpose

This package contains everything Langium generates and everything written on top of it:

- The BPMNscript grammar (`src/bpmn-script.langium`) covering the supported BPMN subset.
- The structural validator (`src/bpmn-script-validator.ts`) for AST-level BPMN constraints.
- The Langium-generated parser, AST types, and language server module (built artifacts in `out/`; source in `src/generated/` â€” both git-ignored, regenerated by `npm run build`).
- The VS Code TextMate grammar and Monarch syntax definition.

## Grammar overview

The grammar accepts files with one `process` block per file:

```bpmnscript
process invoice-approval "Invoice Approval" {
  start ReviewStart
  user  ReviewInvoice "Review invoice" assignee: "demo"
  gateway AmountCheck "Amount > 1000?" default: AutoApprovePath
  user  SeniorApproval "Senior approval" assignee: "manager"
  service AutoApprove "Auto-approve" class: "com.example.AutoApproveDelegate"
  end   Done

  ReviewStart    -> ReviewInvoice
  ReviewInvoice  -> AmountCheck
  AmountCheck    -> SeniorApproval  when: "${amount > 1000}"
  AmountCheck    -> AutoApprove     as: AutoApprovePath
  SeniorApproval -> Done
  AutoApprove    -> Done
}
```

Supported node kinds: `start`, `end`, `user` (optional `assignee:`, `formKey:`), `service` (required `class:`), `gateway` (optional `default:`). Sequence flows accept optional `when:` (condition expression) and `as:` (flow identifier, referenced by `default:`).

## Validator checks

The built-in structural validator in `src/bpmn-script-validator.ts` reports errors for:

- A process with no `start` event.
- A process with no `end` event.
- A sequence flow whose source or target does not resolve to a declared node.
- A gateway's `default:` that does not match any `as:` label among its outgoing flows.
- Orphan nodes (nodes not connected by any sequence flow).

## Build and test

```bash
# From repo root
npm run build --workspace packages/language
npm test --workspace packages/language

# From this directory
npm run build    # runs langium generate + tsc
npm test
```

The `langium:generate` and `langium:watch` scripts are also available for IDE-only use (the VS Code launch configuration relies on them).
